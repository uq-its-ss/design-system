## Overview

The card component is used to provide a brief summary of content or a task, often with a link to more detail. Cards are frequently displayed alongside other cards to group related content or tasks.
The height of the cards in a group all equalises by default. 

## Default example

> By default, the card component is clickable anywhere on the card and when hovered or active has a shadow but when it contains a text link or a button, it is clickable only on the link or button and drops off the shadow.

### Special rules

~ On smaller viewports, image height is reduced to 190px for 1,2 column story/event cards and 1,2,3,4 column default cards. <br />
** On smaller viewports, image height remain fixed at 98px for story/event cards.

## Accessibility

- With keyboard accessibility, tabbing takes the user through the links in the card.
- With assistive technologies like screen readers, it reads out the heading level, title, text and links in the card.
