# Breakpoints

Based on current display trends (as well as standard practice of covering mobile
& tablet devices to desktop/laptop screens), our _mobile-first_ breakpoints are:

- tablet: 37.5rem (~600px),
- and desktop/laptop: 74rem (~1200px).

Full details, including Sass variables as follows:

| Name(s)                 | Constant variable | Default constant     | Media query string variables                          | Default viewport width ranges                                                                    |
| ----------------------- | ----------------- | -------------------- | ----------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| **\* (mobile)**         | N/A               | N/A                  | N/A                                                   | 0—∞                                                                                              |
| _Deprecated: XS_        | $screen-xs        | 26rem **(~416px)**   | $screen-xs-up,<br/>$screen-xs-down,<br/>$screen-xs-sm | 26rem—∞ **(~416px—∞)**,<br/>0—25.9375rem **(~0—415px)**,<br/>26—33.9375rem **(~416—543px)**      |

_Pixel units (px) provided are based on the default rem unit analogue of 16px._

## Usage

We recommend using the breakpoints in a _mobile-first_ pattern. The general idea
is that you set styles for the smallest screens first (e.g. mobile phones) and
then making the modifications that you need for larger screens. Here's an
example:

```scss
/* example.scss */

@use "@uqds/core/src/scss/_global" as core;

.example-class {
  // Target the zero (mobile) breakpoint & up
  font-size: core.$font-size-s;
  padding: core.$space-m;
  text-align: center;
}

@media #{core.$screen-md-up} {
  .example-class {
    // Target the MD (tablet) breakpoint & up
    font-size: core.$font-size;
  }
}

@media #{core.$screen-xl-up} {
  .example-class {
    // Target the XL (desktop) breakpoint & up
    font-size: core.$font-size-m;
  }
}
```
