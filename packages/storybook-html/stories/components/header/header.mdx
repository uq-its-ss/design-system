# Global header

<StatusBadge.Stable />

## Overview

The UQ Global Header provides consistent branding and navigation across all UQ websites. It introduces UQ branding to make sites instantly recognizable and provides a standardized information architecture for navigation.

The header has been rebuilt with improved accessibility, modern interaction patterns, and a modular architecture to ensure reliable performance across all devices.

## How to initialize

Use JavaScript to initialize the header component. This ensures all interactive features (mega menu, mobile menu, search) work correctly.

```js
// Initialize the header
const headerElem = document.querySelector(".uq-header");
if (headerElem) {
  new uq.header(headerElem);
}

// Optional: Set CSS custom property for mobile viewport height
const appHeight = () => {
  const doc = document.documentElement;
  doc.style.setProperty("--app-height", `${window.innerHeight}px`);
};
window.addEventListener("resize", appHeight);
appHeight();
```

The header initialization automatically sets up:

- **Mega menu** interactions (click/keyboard activation)
- **Mobile slide menu** with nested navigation support
- **Search toggle** functionality
- **Responsive behavior** at the 1024px breakpoint

**Implementation notes:**

- When a parent container has `data-gtm-category`, child interactive elements inherit this category unless they specify their own
- Only leaf elements (actual links/buttons) receive `data-gtm-label` attributes

- Hierarchical paths use `>` separator: `"Parent > Child"` (e.g., `"Study > Find a course"`)

## Key parts

### Mega menu

The mega menu provides extended navigation for top-level UQ sections.

**Purpose:**  
Displays comprehensive sub-navigation options in an organized, multi-column layout for high-traffic sections of the UQ website.

**Interaction model:**

- **Desktop (1024px and above):** Click or press Enter/Space on primary navigation items to toggle mega menus
- **Keyboard navigation:**
  - `Enter` or `Space` — Open/close mega menu
  - `Escape` — Close mega menu and return focus to trigger button
  - `Tab` — Navigate through menu items
- **Auto-close:** Opening one mega menu automatically closes others
- **Click outside:** Clicking outside the header closes all open mega menus

**States:**

- Default (closed): `aria-expanded="false"`
- Open: `aria-expanded="true"` with `.uq-header__nav-primary-item--is-open` class

**Accessibility features:**

- ARIA attributes (`aria-expanded`, `aria-haspopup`, `aria-controls`) announce menu states
- Proper `<button>` elements for menu triggers (semantic HTML)
- Keyboard navigation follows logical, sequential focus order
- Screen readers announce menu states and hierarchical relationships

### Mobile menu

The mobile navigation provides a slide-out menu system optimized for touch devices.

**Purpose:**  
Delivers an intuitive, hierarchical navigation experience on mobile devices with support for deep nested menu structures.

**Display:**

- **Mobile:** Viewports 1023px and below
- **Desktop:** Hidden at 1024px and above

**Interaction:**

- Tap the menu toggle button to open/close the mobile menu
- Tap items with chevrons to navigate deeper into menu levels
- Tap the back arrow to return to the previous menu level
- Menu always opens to the section containing the active page
- Scroll position resets to top when navigating between menu levels

**States:**

- The mobile menu highlights the current active page
- When opened, the menu defaults to the active section
- Closing and reopening resets to the active section

**Visual design:**

- Background: UQ purple (`core.$purple-600`)
- Inline SVG icons for back arrows and forward chevrons
- Standardized spacing using design tokens

**Touch device optimization:**

- Sticky hover states are neutralized using `@media (hover: none)`
- Prevents visual artifacts when tapping menu items

### Search

The search component provides access to UQ-wide search functionality.

**Purpose:**  
Allows users to search across UQ websites using [UQ Search](https://search.uq.edu.au).

**Interaction:**

- Click/tap the search icon to toggle the search panel
- Search panel slides down from the header
- Panel closes when clicking outside or toggling the search button again

**Animation:**

- GPU-accelerated transform animations for smooth performance
- Uses `translateY()` instead of `opacity` for better rendering
- Hardware acceleration via `will-change: transform`

**States:**

- Closed: Search panel hidden above the header
- Open: Search panel slides into view below the header navigation

## Accessibility

The header component is built to meet WCAG 2.1 AA accessibility standards.

### Keyboard navigation

**Mega menu controls:**

- `Tab` — Move through navigation items
- `Enter` or `Space` — Open/close mega menu
- `Escape` — Close open mega menu and return focus to trigger
- Arrow keys — Navigate through mega menu items

**Focus management:**

- Visible focus indicators on all interactive elements
- Focus is properly managed when opening/closing menus
- Focus is not trapped — users can always exit menus

### Screen reader support

**ARIA attributes:**

- `aria-expanded` — Indicates whether mega menus are open or closed
- `aria-haspopup="true"` — Indicates elements have dropdown menus
- `aria-controls` — Associates triggers with their corresponding menu panels
- Semantic `<button>` elements for interactive controls without href destinations

**Semantic structure:**

- Proper heading hierarchy
- Semantic HTML elements (`<nav>`, `<ul>`, `<li>`)
- Meaningful link text
- Unique IDs for all menu panels and triggers

### Mobile accessibility

- Touch targets meet minimum size requirements (44×44px)
- Mobile menu supports swipe gestures
- Content remains accessible when zoomed to 200%
- Color contrast meets AA standards (4.5:1 for text)

### Testing

The header has been validated for:

- ✅ Screen reader compatibility (NVDA, JAWS, VoiceOver)
- ✅ Keyboard-only navigation
- ✅ Touch device interaction
- ✅ High contrast mode support
- ✅ Zoom and text scaling

## Responsive behavior

The header adapts to different screen sizes with a breakpoint at **1024px**.

### Desktop (1024px and above)

- Displays horizontal primary navigation with mega menu dropdowns
- Secondary navigation visible in header
- Search panel slides down from navigation bar
- Hover states active (on devices with hover capability)

### Mobile (1023px and below)

- Primary and secondary navigation hidden
- Mobile menu toggle button displayed
- Slide-out menu panel activated
- Search integrated into mobile menu experience
- Optimized for touch interaction

### Performance

- **GPU-accelerated animations** — Transform-based transitions for smooth 60fps animations
- **Efficient event handling** — Uses `matchMedia` API instead of resize events for breakpoint detection
- **Modular architecture** — Clean separation of concerns for better maintainability
- **Lazy initialization** — Components only initialize when elements are present
