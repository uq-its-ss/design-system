## Overview

The `.uq-local-nav` element provides local navigation for desktop.

<Canvas withSource="none">
  <Story id="components-navigation-local-nav--local-nav" />
</Canvas>

### How-to

#### Parents and Grandparents

A page can have multiple `.uq-local-nav__grandparent` (Grandparent) elements,
but only one `.uq-local-nav__parent` (Parent) element.

<Canvas withSource="none">
  <nav className="uq-local-nav" aria-label="Local navigation">
    <div className="uq-local-nav__menu-item uq-local-nav__grandparent">
      <a href="#">
        <span className="uq-sr-only">Back to </span>UQ home
      </a>
    </div>
    <div className="uq-local-nav__menu-item uq-local-nav__grandparent">
      <a href="#">
        <span className="uq-sr-only">Back to </span>Research
      </a>
    </div>
    <div className="uq-local-nav__menu-item uq-local-nav__grandparent">
      <a href="#">
        <span className="uq-sr-only">Back to </span>Research capabilities and
        initiatives
      </a>
    </div>
    <div className="uq-local-nav__menu-item uq-local-nav__parent">
      <a href="#">
        <span className="uq-sr-only">Back to </span>Energy network
      </a>
    </div>
  </nav>
</Canvas>

#### Menu links

After the Parent element, use the following standard `ul`/`li` menu structure.

The active menu item is marked with `.uq-local-nav--is-active` on the `li` element, and
for accessibility the active `a` element is marked with `aria-current="page"`

The current menu section (which can also be the active menu item) is marked with
`.uq-local-nav--is-current` on the `li` element.

Menu items that have children are marked with `.uq-local-nav--has-children` on
the `li` element.

#### Expanded sections

The `ul` / `li` menu structure is only ever two levels deep.

A `ul.uq-local-nav__child-menu` element should be used for the second level.

Child menu items have both `.uq-local-nav__menu-item.uq-local-nav__child` classes

The current section menu item is marked with `.uq-local-nav--is-current` on the
first level `li` element.
