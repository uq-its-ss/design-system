## Overview

The Grid component is a layout utility designed to arrange content in a structured, responsive manner using 1 - 4 columns. It uses CSS Grid logic to automatically manage column wrapping based on available space, ensuring a consistent user experience across all devices without requiring manual media query management.

## Usage Guide

### Multi-Column Layouts

When using 2, 3, or 4-column modifiers, the grid calculates the necessary space by subtracting the gap width from the percentage-based width to ensure a perfect fit.

### For Designers

- **Column Flexibility**: You can choose between 1, 2, 3, or 4-column layouts.
- **Minimum Width**: Each grid item has a "hard floor" of **317px**. If a container becomes too narrow to hold items at that width, they will automatically wrap to a new row.
- **Consistency**: Use the grid to align repetitive elements like Profile Cards, News snippets, or Galleries.

### For Developers

The grid uses the `auto-fit` and `minmax` strategy to create a fluid layout.

- **Base Class**: Use `.uq-grid` on the container.
- **Modifiers**: Apply modifiers like `.uq-grid--3-column` to set the _ideal_ maximum number of columns.
- **Gap Management**: The spacing between items is controlled by the CSS variable `--uq-grid-gap`.

### Best Practices

1.  **Don't Nest Too Deeply**: While the grid is flexible, nesting multiple grids can lead to very narrow columns that trigger the 317px wrap immediately.
2.  **Content Sizing**: Ensure the content inside the grid items can handle a minimum width of 317px gracefully.
